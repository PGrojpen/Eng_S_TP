# Avaliação de Issues (TP1)

## Diagnóstico
No estado atual do repositório local, não há evidência de issues versionadas (apenas README).

## Inconsistências encontradas
- Falta de padronização para criação de issues.
- Falta de vínculo explícito entre issues e histórias de usuário/requisitos.

## Melhorias aplicadas
- Criação de templates de issue para **Feature** e **Documentação** em `.github/ISSUE_TEMPLATE`.
- Inclusão de critérios de aceite e definição de pronto (DoD) para manter simplicidade e rastreabilidade.

## Como usar no GitHub Projects
- Criar as issues a partir dos templates.
- Adicionar todas em **Project Backlog**.
- Mover para **TODO (TP2)** somente as prioridades do próximo sprint.
