# Planejamento Geral e Backlog do TP2 (TP1)

Este documento organiza, de forma simples, o que deve estar no GitHub Projects.

## Colunas recomendadas no GitHub Projects
1. **Project Backlog** (todos os requisitos e histórias)
2. **TODO (TP2)** (o que entra no próximo sprint)
3. **Doing**
4. **Done**

## Roadmap resumido por sprint
- **TP1:** problema, requisitos, casos de uso, planejamento.
- **TP2:** arquitetura (C4), decisões tecnológicas, documentação arquitetural.
- **TP3:** primeiro entregável funcional + vídeo.
- **TP4:** evolução do software + plano de testes (mínimo 3 casos por caso de uso) + vídeo.
- **TP5:** execução dos testes planejados + resultados + vídeo.
- **TP6:** entrega final com requisitos concluídos, documentação revisada e testes consolidados.
